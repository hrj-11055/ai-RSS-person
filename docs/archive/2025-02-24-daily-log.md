### 1. 中文新闻保障机制
**问题**: 日报中中文新闻数量不稳定，有时不足 10 篇

**解决方案**: 在 `article_ranker.py` 中实现带中文配额的排序算法

```python
    ...
]

            return True
    return False

    articles, top_n=50, chinese_quota=10
) -> List[Dict]:
    """确保最终结果中中文新闻 >= chinese_quota"""
    # 1. 正常排序获取候选
    # 2. 分离中文/英文文章
    # 3. 中文不足时从全部文章中补充
    # 4. 优先中文，再填充其他
```

**关键点**:
- 先按质量排序取 2 倍候选 (top_n * 2)
- 分离中英文文章后单独处理
- 中文不足时从原始池中按得分补充
- 最终结果确保中文数量达标

### 3. 移除 Docker 依赖
**发现**: 用户使用 npm 部署 RSSHub (端口 1200)，而非 Docker

**改动**:
- 移除 Docker 容器检查逻辑
- 改用 `curl http://localhost:1200` 健康检查
- 删除 `docker-compose.yml` 中 RSSHub 相关配置

```bash
# 旧逻辑
RSSHUB_RUNNING=$(docker ps -q -f name=rss-person-rsshub)

# 新逻辑
if curl -s -o /dev/null -w "%{http_code}" http://localhost:1200 | grep -q "200"; then
    echo "✅ RSSHub 服务运行正常"
fi
```

**教训**: 不要假设基础设施，先验证实际部署方式

### 4. 创建防休眠包装脚本
**问题**: macOS 长时间任务可能因休眠中断

**解决方案**: 使用 `caffeinate` 命令

```bash
#!/bin/bash
# run_daily_report_with_caffeinate.sh

# caffeinate -s 会阻止系统休眠，直到命令完成
caffeinate -s "$PROJECT_DIR/scripts/mac/daily_report_with_docker_check.sh"
```

**launchd 配置关键点**:
```xml
<key>ProcessType</key>
<string>Interactive</string>  <!-- 允许在后台运行但保持活跃 -->
```

### 5. 清理重复定时任务
**发现**: 两个 launchd 任务都在 07:00 执行日报
- `com.aicoming.dailyreport` (有防休眠)
- `com.aireport.daily` (直接执行)

**解决**: 卸载 `com.aireport.daily`，保留带防休眠保护的版本

```bash
launchctl unload ~/Library/LaunchAgents/com.aireport.daily.plist
rm ~/Library/LaunchAgents/com.aireport.daily.plist
```

## 附录：RSS 源配置与去重机制

### RSS 源接入流程
使用 Follow 应用管理 RSS 订阅，一键导入 OPML 文件后，只需配置 JSON 字段：

```yaml
# config/sources.yaml 配置模板
sources:
  - name: "源名称"
    url: "RSS链接"
    strategy: "direct|rsshub|noproxy"
    enabled: true
    category: "分类"
```

**策略说明**:
- `direct`: 直连抓取（官方博客、科技媒体）
- `rsshub`: 通过 RSSHub 路由（Twitter、YouTube、中文媒体）
- `noproxy`: 不走代理（学术源 Arxiv）

### 文章去重机制
**核心算法**: Jaccard 相似度 + 源权重优先

```python
# lib/article_dededucator.py
def deduplicate(articles):
    # 1. 标题清洗（移除标点、网站名）
    # 2. 分词处理
    # 3. Jaccard 相似度 = |A∩B| / |A∪B|
    # 4. 相似度 ≥ 0.85 视为重复
    # 5. 保留高权重源的文章
```

**去重流程**:
```
原始文章 (~200-400篇)
    ↓
按源权重排序（高→低）
    ↓
逐篇检查相似度
    ↓
┌─────────────────────────────────┐
│  文章A (机器之心, 权重90)         │
│  文章B (量子位, 权重96)           │  → 保留量子位
│  文章C (新智元, 权重97)           │  → 保留新智元
└─────────────────────────────────┘
```
